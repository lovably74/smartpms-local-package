# WBS 테스크 진행상태·진행률·실행일

## 테스크

- 진행상태는 **미시작 / 진행중 / 완료** 중 하나를 드롭다운으로 선택한다.
- 선택 시 진행률은 각각 **0% / 50% / 100%**로 저장된다.
- **미시작**이면 실행일(`actualStart`)은 비우며, 화면 실행일 열에는 값을 표시하지 않는다.
- **진행중·완료**로 바꾼 경우 실행일은 **변경한 날짜(서버 기준 당일)**로 저장된다.

## 진행률 직접 수정

- 기존처럼 수치로 진행률을 바꿀 수 있다.
- **0%**로 두면 진행상태는 **미시작**, 실행일은 초기화된다.
- **100%**로 두면 진행상태는 **완료**, 실행일은 해당 저장 시점의 날짜로 갱신된다.
- 0%에서 0% 초과로 처음 올리면 실행일을 그때 날짜로 설정한다(이미 진행 중인 경우 일부 구간만 조정할 때는 실행일 유지).

## 상위 공종(액티비티·소·중·대공종)

- 진행률: 직접 자식 진행률의 **산술 평균**(기존과 동일).
- 진행상태 롤업:
  - 자식이 **전부 완료** → 완료
  - **전부 미시작** → 미시작
  - 그 외(일부만 완료·진행중 혼합, 또는 하나라도 진행중 등) → **진행중**  
  (DB에 남아 있는 지연·보류 값은 롤업 시 **진행중**으로 취급)
- 실행일: 직접 자식 실행일 중 **가장 늦은 날짜**를 상위 실행일로 저장한다(자식이 모두 비어 있으면 null).

## WBS 그리드 2행 필터(전체·진행중·지연·미시작·완료)

- **보류** 필터는 없다.
- **전체**가 아닌 필터를 선택한 경우: 조건에 맞는 **테스크**만 대상으로 하고, 그 테스크에서 **상위로 이어지는 공종**(액티비티·소·중·대공종)만 행으로 표시한다.
- 조건에 맞는 테스크가 **하나도 없으면** 목록은 비운다(상위 공종도 표시하지 않음).

## WBS 화면 상단 「지연」 통계·필터

- **지연** 건수: **현재 날짜(브라우저 로컬)** 기준으로 **계획종료일이 오늘보다 이전**이고, 진행상태가 **완료가 아닌** 테스크 수.
- 계획종료일이 없으면 지연에 포함하지 않는다.
- 같은 날이 계획종료일인 경우는 지연으로 보지 않는다(종료일 다음 날부터 지연).

## API

- `wbs.setTaskProgressStatus`: 테스크만, `{ id, projectId, status: 'not_started' | 'in_progress' | 'completed' }`
- `wbs.updateProgress`: 진행률 저장 및 테스크 실행일·진행상태 연동(위 규칙).
